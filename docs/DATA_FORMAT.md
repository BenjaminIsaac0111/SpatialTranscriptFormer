# SpatialTranscriptFormer Data Guide

`SpatialTranscriptFormer` is designed to be a flexible foundation model for spatial transcriptomics. While it provides an automated downloader for the HEST dataset, **you can bring any custom Spatial Transcriptomics dataset** as long as it adheres to the standard `AnnData` (`.h5ad`) format.

This guide covers both how to format your own custom data, and how the built-in HEST dataset is structured.

---

## 1. Using Custom Data

### Directory Structure

The package expects a specific directory structure within your configured `data_dirs` (see `config.yaml`). If you are providing pre-computed features (the default), your data directory should look like this:

```text
your_data_dir/
├── st/                           # Raw .h5ad files containing gene expression
│   ├── sample1.h5ad
│   ├── sample2.h5ad
│   └── ...
├── he_features_ctranspath/       # Pre-computed patch features
│   ├── sample1.pt
│   ├── sample2.pt
│   └── ...
└── global_genes.json             # Generated by stf-build-vocab
```

*Note: If you are training from scratch on raw image crops, you would have a `patches/` directory with `.h5` files instead of the `he_features_ctranspath/` directory.*

### AnnData (`.h5ad`) Requirements

The files in the `st/` directory must be valid `AnnData` objects saved using `scanpy` or `anndata`. The package looks for specific keys to extract barcodes, genes, counts, and spatial metadata.

If you processed your data using standard tools like **10x SpaceRanger** and loaded it into Python via `scanpy.read_visium()`, your data is likely already in the correct format!

#### `obs` (Observations / Spots)

- Must contain an index representing the unique barcodes for each spot/cell.
- The index is used to align the gene expression vectors with the spatial features.

#### `var` (Variables / Genes)

- Must contain an index representing the gene names (e.g., standard HGNC symbols like `TRAP1`, `BRCA1`).
- These names are used by `stf-build-vocab` to map your dataset to biological pathways (like MSigDB Hallmarks).

#### `X` (Expression Matrix)

- Can be either a dense matrix or a sparse CSR matrix.
- Should contain the **raw counts** (not log-normalized or scaled, as the model applies its own transformations like `log1p` if configured).
- Shape must be `(n_obs, n_vars)`.

#### `uns` (Unstructured Metadata)

The model needs spatial metadata to map coordinates to the original histology image. This must follow the standard 10x spatial format:

```python
adata.uns['spatial'] = {
    'your_sample_id': { # The model will pick the first key here
        'images': {
            'hires': np.ndarray,   # Shape (H, W, C) - The histology image
            # or 'lowres': np.ndarray
        },
        'scalefactors': {
            'tissue_hires_scalef': float, # The scale factor to map coordinates to 'hires'
            # or 'tissue_lowres_scalef': float
        }
    }
}
```

*Note: `SpatialTranscriptFormer/src/spatial_transcript_former/data/io.py` is quite resilient. It will look for `hires` first, but if your keys are named differently (e.g., `downscaled_fullres`), it will attempt to find the first available image and matching scalefactor.*

### Workflow for Custom Data

Once your files match the structure above:

1. **Place data:** Put your `.h5ad` files into `your_data_dir/st/`.
2. **Build Vocabulary:** Run `stf-build-vocab --data-dir your_data_dir/`. This will scan all your `.h5ad` files, find the most highly expressed genes, map them to biological pathways, and generate `global_genes.json`.
3. **Extract Features (Optional):** If you haven't already, run the feature extraction pipeline (e.g., `stf-extract`) to generate the `.pt` files in `he_features_ctranspath/`.
4. **Train:** You are now ready to run `stf-train`!

## 2. Default HEST Dataset Structure

If you use the built-in downloader (`stf-download`), the HEST (Heterogeneous Evaluation of Spatial Transcriptomics) dataset will be organized on disk as follows:

```text
hest_data/
├── HEST_v1_3_0.csv        # Central metadata registry
├── README.md              # Dataset documentation from source
├── patches/               # H5 files containing image patches and local coordinates
│   ├── MEND29.h5
│   └── ...
├── st/                    # H5AD files containing gene expression data
│   ├── MEND29.h5ad
│   └── ...
├── wsis/                  # Whole Slide Images (WSIs) in .tif or .svs format
├── cellvit_seg/           # Nuclear segmentation maps (processed from CellViT)
├── spatial_plots/         # Visualization plots (optional)
└── ...
```

### Core HEST Components

1. **Metadata Registry (`HEST_v1_3_0.csv`)**
   The source of truth for all samples. It contains:
   - `id`: Unique identifier for the sample (e.g., `MEND29`).
   - `organ`: Primary organ (e.g., `Bowel`, `Breast`).
   - `species`: Species (e.g., `Homo sapiens`).
   - `st_technology`: Technology used (e.g., `Visium`, `Xenium`).
   - `patient`: Patient ID for stratified splitting.

2. **Patches (`patches/*.h5`)**
   Each H5 file corresponds to one sample and contains:
   - `img`: `(N, 224, 224, 3)` array of image patches (uint8).
   - `coords`: `(N, 2)` array of local (x, y) coordinates for each patch.
   - `barcode`: `(N, 1)` array of spot barcodes corresponding to the ST data.

3. **Spatial Transcriptomics (`st/*.h5ad`)**
   The transcriptome data preserved in AnnData format:
   - `X`: Sparse or dense matrix of gene expression counts.
   - `obs`: Observations metadata (containing barcodes in `_index` or `index`).
   - `var`: Variable metadata (containing gene names in `_index` or `index`).

4. **Segmentation & WSIs**
   - **WSIs**: High-resolution scans found in `wsis/`.
   - **Segmentation**: Pre-computed nuclei masks found in `cellvit_seg/` or technology-specific folders like `xenium_seg/`.
